Introduction

JDev Mail API provides a simple, RESTful interface for sending professional emails from your applications.

Base URL 
https://api.jdevmail.com

Authentication

All API requests require authentication using your site credentials.

Authentication Headers 
{
  "site_id": "your_site_id",
  "public_key": "your_public_key",
  "secret_key": "your_secret_key"
}

Send Email

Send an email through the API.

POST /api/send-email

Request Body

{
  "site_id": "string (required)",
  "public_key": "string (required)",
  "secret_key": "string (required)",
  "to": "string (required, email)",
  "subject": "string (required)",
  "html_content": "string (required)",
  "text_content": "string (optional)"
}

Example Request

cURL 
curl -X POST https://api.jdevmail.com/api/send-email \
  -H 'Content-Type: application/json' \
  -d '{
    "site_id": "your_site_id",
    "public_key": "your_public_key",
    "secret_key": "your_secret_key",
    "to": "user@example.com",
    "subject": "Welcome to JDev Mail",
    "html_content": "
Welcome!
"
  }'

Response

{
  "success": true,
  "message": "Email sent successfully",
  "log_id": 123
}

Python Integration

Example using Python with requests library.

Python 
import requests

url = 'https://api.jdevmail.com/api/send-email'
data = {
    'site_id': 'your_site_id',
    'public_key': 'your_public_key',
    'secret_key': 'your_secret_key',
    'to': 'user@example.com',
    'subject': 'Test Email',
    'html_content': '
Hello World
'
}

response = requests.post(url, json=data)
print(response.json())

Error Codes

Code Description
400Bad Request - Missing or invalid parameters
401Unauthorized - Invalid authentication credentials
403Forbidden - Rate limit exceeded or account locked
404Not Found - Resource not found
429Too Many Requests - Rate limit exceeded
500Internal Server Error
